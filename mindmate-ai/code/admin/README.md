# MindMate AI Admin Dashboard

A comprehensive admin dashboard for the MindMate AI platform built with Next.js 14, TypeScript, and Tailwind CSS.

## Features

### 1. User Management Table
- **Search & Filter**: Search users by name/email, filter by role, status, and subscription tier
- **Sortable Columns**: Sort by any column (name, role, status, sessions, etc.)
- **Pagination**: Navigate through large user lists efficiently
- **User Details Modal**: View comprehensive user information
- **Edit & Delete**: Full CRUD operations with confirmation dialogs
- **Visual Indicators**: Avatar initials, status badges, crisis flag counts

### 2. Crisis Alerts Panel
- **Real-time Updates**: Simulated WebSocket connection for live alerts
- **Severity Levels**: Critical, High, Medium, Low with color coding
- **Alert Types**: Self-harm, Suicide ideation, Violence, Substance abuse, Other
- **Acknowledge Workflow**: Acknowledge, escalate, or resolve alerts with notes
- **Status Tracking**: New, Acknowledged, In Progress, Resolved, Escalated
- **Statistics Dashboard**: New alerts count, critical alerts, connection status

### 3. Session Flags Review Interface
- **AI-Detected Flags**: Review flags generated by AI monitoring
- **Flag Types**: Concerning behavior, Crisis indicators, Quality issues, Boundary violations
- **Review Workflow**: Review, resolve, or dismiss flags with documentation
- **AI Confidence Scores**: Visual indicators of AI detection confidence
- **Grid/Table Views**: Toggle between visual card layout and data table
- **Filtering**: Filter by status, severity, and flag type

### 4. Usage Analytics Charts
- **Interactive Charts**: Line, bar, and area chart types
- **Key Metrics**: Active users, new users, sessions, messages, avg duration, crisis alerts
- **Time Ranges**: 7 days, 30 days, 90 days views
- **Metric Toggles**: Show/hide different metrics on the chart
- **Sparklines**: Mini trend charts in the data table
- **Daily Breakdown**: Detailed table with all metrics per day

### 5. Notification Broadcast Tool
- **Create Broadcasts**: Compose notifications with title, message, type, and target audience
- **Scheduling**: Send immediately or schedule for later
- **Target Audiences**: All users, users only, therapists only, admins only, custom filters
- **Notification Types**: Info, Warning, Critical, Maintenance
- **Delivery Tracking**: Monitor sent count and failed deliveries
- **Preview Mode**: Preview notifications before sending

### 6. System Health Monitor
- **Real-time Monitoring**: Live service status updates
- **Service Grid**: Visual overview of all system services
- **Health Metrics**: Uptime, response time, error rate per service
- **Status Indicators**: Healthy (green), Degraded (yellow), Down (red)
- **Response Time Chart**: Visual comparison of service response times
- **24-Hour History**: Simulated status history visualization
- **Service Actions**: Restart service, view logs (UI ready)

## Tech Stack

- **Framework**: Next.js 14 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **UI Components**: Custom component library
- **State Management**: React hooks (useState, useCallback, useMemo)
- **Real-time**: Simulated WebSocket with useEffect intervals

## Getting Started

### Installation

```bash
# Install dependencies
npm install

# Run development server
npm run dev

# Build for production
npm run build
```

### Environment Variables

Create a `.env.local` file:

```env
# Optional: API endpoints for real data
NEXT_PUBLIC_API_URL=http://localhost:3000/api
NEXT_PUBLIC_WS_URL=wss://api.mindmate.ai/admin/realtime
```

## Component Usage

### User Management Table
```tsx
import { UserManagementTable } from '@/components/UserManagementTable';

<UserManagementTable 
  users={users} 
  onUserUpdate={(user) => console.log('Update:', user)}
  onUserDelete={(userId) => console.log('Delete:', userId)}
/>
```

### Crisis Alerts Panel
```tsx
import { CrisisAlertsPanel } from '@/components/CrisisAlertsPanel';

<CrisisAlertsPanel initialAlerts={alerts} />

### Session Flags Review

import { SessionFlagsReview } from '@/components/SessionFlagsReview';

<SessionFlagsReview initialFlags={flags} />

```

## Customization

### Adding New Metrics to Analytics

Edit `types/index.ts` to add new metric types:

```typescript
export interface UsageMetrics {
  date: string;
  activeUsers: number;
  // Add new metrics here
  newMetric: number;
}
```

### Customizing Theme Colors

Edit `tailwind.config.js`:

```javascript
colors: {
  primary: {
    500: '#your-color',
  },
}
```

### Connecting to Real API

Replace mock data imports in `page.tsx`:

```typescript
// Replace this:
import { mockUsers, mockCrisisAlerts } from '@/data/mockData';

// With API calls:
const users = await fetch('/api/users').then(r => r.json());
```
